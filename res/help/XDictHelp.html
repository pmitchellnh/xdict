  1. Overview

    2. XDict is a database tool designed to facilitate the creation and maintenance of a crossword puzzle dictionary. It was designed and continues to be maintained by a crossword puzzle constructor for personal use, and its features are born from the actual needs and pains of trying to maintain word lists through existing crossword construction tools (Crossword Compiler, Crossfire) or other non-crossword-specific applications (Word, Excel, text files).

  2. Features

    • Supports entries between 3 and 25 letters long
    • Full-featured query, including "wildcard" letters
    • Load rated or unrated word lists
    • Quick-and-easy method of hand-rating words
    • Track which words have been hand-rated
    • Track words that have been used in NY Times (and other publications)1
    • Reconcile logic handles "smart" merging of new lists into existing database without affecting words that have already been hand-rated
    • Add comments of up to 100 characters, for clarifications, notes, or clue suggestions
    • Optional adjusted rating algorithm based on word length2
    • Export word lists appropriate for Crossword Compiler or Crossfire
    • Backup/restore the database to/from file
    • View rating statistics, broken down by word length
    • "Test" mode allows you to experiment using a temporary database without affecting your production dictionary
    • Configuration file allows you to customize certain key behaviors

    1Requires manual entry or proper setup when loading certain word lists.
    2Based on my own personal rating theory. See Rating Words for more details.

  5. A Tour of the User Interface

    1. Menu Bar Commands

      2. Certain actions can be done from the menu commands. They are as follows:

      • XDict Menu: About this software.
      • View Menu: Reset parameters to the default for the selected action.
      • Reports Menu: View rating progress and breakdown reports.
      • Database Menu: View table info; backup, restore, and delete tables.
      • Help Menu: View this documentation.

    2. Metadata Parameter Settings

      3. These settings are used to query on or set various metadata parameters. Not all parameters are applicable in certain modes. How each is used in each mode is discussed in detail in the appropriate sections.

    3. Action Buttons

      4. Each action mode has different action buttons associated with it. The default action (which occurs if you just hit the Enter key) at any given time is the button that is colored blue.

      Note:

      The default action may change as appropriate, as described in the sections below.

    4. Action Mode Tabs

      5. The Action Mode Tabs control the type of major action you are doing. Each action will set certain default parameter values, based on how one is most likely to use that mode. See the sections on each action for details. The action modes are:

      • Query: Find entries in your database based on specific criteria.
      • Add: Manually add, modify, and/or rate single entries to your database.
      • Rate: Rate lists of words quickly, using preset buttons.
      • Load: Load rated or unrated word lists into your database.
      • Export: Create dictionary files to use with your constructing software.

    5. Display Area

      6. The display area is where query results and other messages are shown, and where the rating takes place. It becomes a scrollable region when necessary.

    6. Status Bar

      7. The status bar shows the progress and status of actions.

  6. Querying the Database

    7. The Query tab provides a robust query interface to view your database. All of the metadata parameters can be used to constrain the query, with the exception of "Comments". There is no current mechanism to query based on the contents of the Comments field.

    For large queries, be aware that the system only returns up to 1000 entries per request. For queries that have more than 1000 entries in the result set, use the "Next" button (which will become the default action) to view the next 1000 entries. You can continue using the "Next" action until the result set is empty. Note that your progress through the list is displayed in the status bar.

    1. Entry Control

      2. The Entry control allows you to constrain the result to match a particular pattern of letters. You can specify whether you want the pattern to exactly match (Equals), match the beginning of (Starts with), or match anywhere in (Contains) an entry in your database.

      Note that, while database entries are all stored in upper case, the entry field is not case-sensitive.

      Tip:

      The entry field will accept '?' or '_' characters as single-character wildcards. For example, an Equals query on "a?t?d" (or "a_t_d") might match ACTED, ANTED, or APTED.

      Tip:

      Any query for an exact entry string (an "Equals" search of at least one character with no wildcards) will search for that string ignoring any other parameters that may be set. This is because when you just want to know whether a word is or is not in your database, you don't want to have to guess what the other parameters might or might not be.

    2. Length Control

      3. The Length control allows you to constrain the result to match a particular length, a minimum length, or a maximum length. The default is "at least 3", which returns all lengths (since the database does not store entries less than 3-letters long). Other examples: "At most 15" would show entries between 3- and 15-letters long, while "Equals 7" would show only entries that are exactly 7-letters long.

    3. Rating Method Control

      4. The Rating Method control allows you to select based on how words have been rated. The software supports three tiers of rating:

      Auto-loaded
      Words loaded with a default value from unrated word lists (or with an invalid or unparseable value from an otherwise rated word list). These words have the lowest rating confidence, as they have not been individually rated.
      Rank-loaded
      Words loaded with values from rated word lists. These entries are considered to have higher confidence than auto-loaded words, as someone has at least attempted to assign them values individually.
      Hand-rated
      Words that you have manually rated by hand. These entries have the highest rating confidence, as you have curated them personally.

      The default query setting is "All".

      Tip:

      You can designate a loading word list as "hand-rated", for cases where you are loading an already manicured list for which you already have highest confidence. See Loading Word Lists for details and warning.

    4. Rating Control

      5. The Rating control allows you to select only entries that are at least, at most, or equal to a particular rating value. This is useful when looking for highly-rated words that match a particular pattern, for instance. When in the "Rating" mode, it can be used to target words that are currently highly or poorly rated. The default setting for queries is "at least 1", which displays everything except "killed" (rating = 0) words.

    5. Crossword Use Control

      6. The Crossword Use control allows you to select entries based on whether they have been used in the New York Times crossword puzzle* (Used NYT) or other "major" crossword puzzles* (Used Any). The default for queries is all entries. The possible combinations are as follows (though not all are necessarily useful):

      *Assuming you have captured this data -- see "Tip" below!

      • All entries.
      • Entries used in any puzzle.
      • Entries used in NY Times puzzle.
      • Entries not used in any puzzle.
      • Entries not used in a NY Times puzzle.
      • Entries not used in NY Times, but used in some puzzle.
      • Entries used in NY Times or not at all.
      • No entries.

      Tip:

      These fields are set when you add (manually) or load word lists, so in order to use them effectively, you must make sure to set the "Used_NYT" field when importing NY Times word lists and the "Used_Any" field when importing other crossword puzzle lists. See Loading Word Lists for details.

    6. Needs Research Control

      7. This field is used to select entries that you flagged as "Research" during the rating process. Note that this constraint is in addition to any other fields that are set, so to get a complete list of entries that need to be researched, use the "Reset Query" action in the "View" menu before querying with this field.

      See the Research and Skip section of The Rating Process for more on using the "Research" field.

    7. Comment Control

      8. This field is not used as a selection criteria (there is currently no mechanism for searching based on the contents of a comment). However, any query that results in a single entry will display that entry's comment in this field (if there is one). Note that comments are limited to 100 characters.

    8. Query and Next Buttons

      9. Pressing the "Query" button (or hitting enter when the button is blue) executes a query based on the metadata parameters set, as described above.

      For queries that result in more than 1000 entries, only the first 1000 will be returned, after which the "Next" button (which will now become enabled and also become the default action) will fetch the next 1000, and so on until there are no more entries that satisfy the criteria.

      Note:

      Note that any time the "Query" button is used, the query will start again from scratch. Also, changing any of the metadata criteria will disable the "Next" button and only allow a fresh query using the new parameters.

  7. Adding and Modifying Words

    8. The Add tab allow you to quickly add or modify (and rate) single entries in your database. When adding a new word manually, you have no choice but to mark it "Hand-rated". Likewise, you cannot specify the length, as that will be calculated after the word is cleared of all non-alphanumeric characters and converted to uppercase. The default value for the Crossword Use control is "Not Used", as typically that is the case when adding new words manually, but you can change this value, add a comment, or select or designate a rating value (default preset is "OK" value), by setting them on the screen before pressing the "Add" button.

    Note:

    Comment lengths are limited to (and will be truncated to, if necessary) 100 characters.

    Note that, unlike in the Rating process, pressing a rating button will not immediately cause the word to be added. The rating buttons are a convenience to set the Manual Slider based on your rating scale. No action to the database will occur until you press the "Add" button. Once the word is added, it will display the formatted word and rating in the Display Area.

    Adding an entry that already exists will not fail; it will modify the existing word with the parameters specified. In fact, as you are typing a word on the Add screen the software will query and update the existing parameters if that word (or those letters as they are currently typed) alreadty exists. It also changes the "Add" button text to "Modify". This is done to prevent accidentally overwriting and erasing metadata from words that already exist without having to do a manual query first -- if the word exists, then the data on the screen matches what is there. Of course, you can then change any of this data and press the "modify" button to override it*.

    *Obviously you cannot change length, as that is inherent to the word. Also, the Rating field in the upper section displays the existing rating -- changes to the rating are made with the rating buttons or Manual Rating slider. This way, the existing rating will continue to be displayed, even while hitting rating buttons, until you have pressed the Modify button (or have left or changed the screen).

  8. Rating Words

    1. Rating Overview and Theory

      2. A lot of thought has been put into how to rate words quickly, efficiently, and intuitively. Rating words is certainly not an exact science -- different people have different ideas about what determines the quality of an entry. Even one's own judgement may be inconsistent from day to day, based on mood or interpretation of a given entry. This is especially true if one is assigning numerical scores. As such, this software pushes the numerical values into the background (after being preset through configuration), allowing the user (you) to rate based entirely on your perception of the entry. It does this by offering seven possible ratings* for each word, as follows:

      • Excellent: These are marquee entries that you would go out of your way to use.
      • Good: These are solid words and phrases that you would be happy to use.
      • OK: These are common words and phrases that may lack pizzazz, but there's nothing wrong with them.
      • Meh: These are below-average or slightly-flawed words or phrases that are okay, but not preferred. This might include certain plurals or gerunds, common abbreviations, common foreign words, common fill-the-blanks, or decent words that are severely limited in how they can be clued.
      • Lame: These are words and phrases that you know you'll need, but want to minimize. I include many abbreviations, foreign words, fill-in-the-blank partials, and proper names that require specific/trivial knowledge, are of another generation, and/or would be difficult to deduce without very clear crossings.
      • Poor: These are obscure entries that you would normally avoid, but might resort to in a pinch.
      • Nope: This is used for non-words or words you have no intention of ever using. Sometimes a sequence of letters shows up in a word list from a rebus, a backwards entry, or some other quirky puzzle that was not filtered out. This can also be used to remove profanity, overly-scientific, obscure variations, etc. This will assign the word a value of 0.

      *The rating values can vary by word length; see below. You can also rate a word manually, if you need more granularity for a given word.

      My theory on word length:

      I don't think it's a stretch to say that long entries carry a puzzle. That is not to say that shorter entries are unimportant; they certainly matter. But the long entries are the ones that your solvers (and editors) are going to notice. Put another way, good long entries are better than good short entries, and bad long entries are worse than bad short entries. There is not a 3-letter entry in existence that is so good that it can make up for a bad long entry. It is not a huge exaggeration to say that almost any 3-letter entry could be justified in a pinch. To put it in mathematical terms, the standard deviation for long entries should, in my opinion, be considerably larger than for short entries.

      This is why this software allows for the configuration of separate rating values for entries that are 3, 4, 5, and 6 letters long. You do not have to subscribe to this theory in full, or at all, but you should think about what you want these values to be before you start any serious manual rating. Note that any config value that is left commented out will default to the non-length-specific value.

      See Configuration for details on setting rating values.

    2. The Rating Process

      3. The query parameters for the Rate action work exactly the same as with the Query action, with the exception that the default value for the Rating Method is "Auto-loaded". This is because you are typically wanting to rate entries that have not yet been hand-rated.

      Also, whereas a regular query returns up to 1000 entries at a time, a rating query is limited to 20 entries per request. This is because rating 1000 entries at once would be a bit overwhelming, so breaking it into 20-entry "chunks" is more conducive to rating several of those chunks in a single sitting. Also, 20 entries are easily displayed without the need to scroll.

      Tip:

      Hand-rating all your words can seem like an impossible task. My advice, as a constructor, is to start small (literally). 3-letter words are the most common, so start there. Then do 4-letter words. This can be a task that you can perform whenever you have a little free time. Got 5 minutes? Rate 100 words.

      When a rate query has returned (up to) 20 results, they will be displayed in the Display Area (along with their current, auto-loaded rating). At this point, you can rate them one by one using the rating buttons. As you rate each one, it will disappear from the top of the list and the next entry will move up. This allows very quick rating, as you need only hit a single button per word.

      Tip:

      You can add a comment to a word during the rating process. Simply type the comment in the "Comments" field before selecting the appropriate rating button.

      The following screen shot is a sample result of a rating query for unrated words that have been in a NY Times puzzle and start with the string "TEST". Note in the status bar that the query has returned the first 20 of a total of 52 entries that meet this criteria.

      If, for whatever reason, you feel like you need more granularity than the rating buttons are providing, you can set any value (from 0 to 100) using the Manual Rating slider and assign it with the "Manual" button.

      Tip:

      When you're filling a themed puzzle, there will be many spots where the fill choice is constrained by the theme entries (typically, down entries crossing the long theme entries). Use the pattern-matching feature to hand rate those patterns before trying to fill the grid. For example, you might need the pattern "??N??C?" or "?X???".

      In this way, your rating activity is specifically targeted to the puzzle(s) you are working on, so you get immediate benefit.

      I typically do this multiple times throughout my fill process: rate words, export to file, import new dictionary, continue filling. Rinse and repeat.

    3. Research and Skip

      4. For any entry during the rating process, you can choose to skip it for now. In this case, you just move on to the next word while making no changes to the skipped word. Both the "Skip" and "Research" buttons skip the current word; the difference between the two is that the "Research" button also flags the entry as needing research. This can be especially useful if you are rating offline (like on an airplane) or are just focusing on quick and easy ratings and want to come back to certain words when you have more time and opportunity to look them up.

      Words that are flagged for research will be automatically cleared of that flag once they are hand-rated.

  9. Loading Word Lists

    10. One of the great features of this software is that is can load word lists into your dictionary without overwriting words and data that you have already curated. See Understand the Reconciler for details about how this works.

    IMPORTANT:

    It is HIGHLY recommended that you backup your database before loading a word list. By backing up first, if the merge goes awry (perhaps you had the wrong parameters set, for example) you can clear the tables, restore to your backup file, and try again.

    The default Rating Method value for the Load screen is "Auto-loaded". This setting covers both unrated and rated word lists, as the software will automatically set "ranked" for any word that is loaded with a value assigned in the word list. See LOAD_FILE_DELIMITERS in Configuration (or comments in the config.txt file itself) for more information about parsing ranked word lists.

    Tip:

    You can choose to load with the "Hand-rated" setting if you are loading an already manicured and rated list, but be certain that's what you want to do before proceeding as all the words in the list will be flagged as "Hand-rated" and treated as such during the merge (see Understand the Reconciler for more about this).

    The default value for Crossword Use is "Not Used". If you are loading a NY Times puzzle word list, then select "Used NYT" so all incoming words will be flagged appropriately. Similarly, if you are loading a multi-puzzle word list, select "Used Any" to flag incoming entries as being used in some puzzle.

    Note: Selecting "Used NYT" implies "Used Any", since the NY Times is a subset of all puzzles, so selecting "Used NYT" forces "Used Any" to also be selected. Conversely, deselecting "UsedAny" will also deselect "Used NYT", since again you cannot specify "NYT" without also specifying "Any". Deselecting "Used Any" will also force "Not Used" to be selected (and vice versa), since entries either are or are not used -- they cannot be both nor can they be neither.

    Tip:

    It is up to you how you want to flag your word lists. The web site Cruciverb.com is a fantastic resource for constructors and includes, as part of its "Gold" membership, access to the puzzle database and two current word lists: "nyt.txt" (NY Times Puzzles) and "all.txt" (NY Times and other major puzzles, including LA Times, Washington Post, Boston Globe, Wall Street Journal, CrosSynergy Syndicate, etc.). I use "Used NYT" when loading nyt.txt and "Used Any" when loading "all.txt".

    The Rating slider is used to specify the default rating of any unrated words that are being loaded. This applies to all words if the list is unrated, but also applies to any unrated words found while processing a rated word list.

    Tip:

    Just because a word has been used in the New York Times or other puzzle, does not necessarily make it a good word or desirable. Many times a puzzle is accepted based on its overall quality that may have an entry or two that frankly isn't that good but is let slide.

    Conversely, just because a word or phrase has NOT appeared in a puzzle, doesn't mean it isn't desirable. Part of making good crosswords is finding those quality words and phrases that haven't been used yet.

    Before loading a word list, make sure the parameters are set the way you want them (including the default rating value). To load a word list, you can either specify the path and file name of the word list in the "File to Load" field" and press the "Load" action button, or select "Browse" and choose the word list from a file select dialog. Note that the default path displayed can be set or changed the in the config.txt file with the LOAD_FILE_DEFAULT_DIR parameter. See Configuration (or the comments in the config.txt file itself) for details.

    Note:

    The default button action in the Load screen is "Browse" unless the path in the "File to Load" field points to an existing file, in which case the default button action will become "Load".

    1. Unranked Word Lists

      2. Unranked word lists are simply lists of words and phrases, one entry per line. It makes no difference whether the lists are in upper-, lower-, or mixed-case, nor whether they include punctuation marks (see Important Note, below). All entries, when loaded, will be stripped of non-alphanumeric characters and converted to all uppercase.

      New entries (those that do not already exist in your database at the time of loading the word list) will be assigned the value set in the Rating slider on the Load screen. Duplicate entries will be merged based on the rules described in the Understanding the Reconciler section.

      IMPORTANT:

      Alphanumeric characters include the letters A to Z and the numerals 0 to 9.

      Please note that the load process will not attempt to convert accented or otherwise diacritically marked letters to their stripped form. These characters, and any other Unicode or non-ASCII alphanumerics, will be considered punctuation and stripped out. If you intend to load a word list that has these characters, you will need to make any letter conversions manually before you load the list.

    2. Ranked Word Lists

      3. Ranked word lists are lists of words and values, with the word and value separated (typically) by a ":" or ";". Note that you can add or change separator values using the LOAD_FILE_DELIMITERS in the config.txt file. See Configuration (or the comments in the config.txt file itself) for details.

      As with unranked word lists, all entries, when loaded, will be stripped of non-alphanumeric characters and converted to all uppercase. Again, see "Important Note" in Unranked Word Lists, above.

      New entries (those that do not already exist in your database at the time of loading the word list) will be assigned the value set in the word list (if it exists) and marked as "ranked", else it will default to the value of the Rating slider on the Load screen (and not be marked as "ranked"). Duplicate entries will be merged based on the rules described in the Understanding the Reconciler section.

      IMPORTANT:

      Importing a ranked word list assumes values in the range of 0 to 100.

      Any value over 100 will be changed to 100.

      Also, any value of 0 will be preserved, so that if you are importing a hand-curated list, entries that you have "killed" (assigned 0 to) will remain so. This is very different than an entry in the list having no value (or no valid non-negative numeric value), in which case the entry will receive the default value for the load.

    3. Understanding the Reconciler

      4. The reconciler is the heart of what allows you to import new lists without adversely affecting your existing dictionary. It kicks in whenever a word loading from an "Auto-loaded" word list already exists in your database. There are essentially three "tiers" of priority: hand-rated, rank-loaded, and auto-loaded. See Rating Method Control for more details.

      Database fields reconcile as follows:

      Rating:
      A new hand-rated word's rating overrides any existing rating.
      An auto-loaded or rank-loaded word's rating will not override a hand-rated rating.
      A rank-loaded word (from a ranked word list) will override an existing auto-loaded rating.
      An auto-loaded word (from an unranked word list) will not override a rank-loaded rating.
      If both incoming and existing words are rank-loaded, the lower rating will prevail. This is done to preserve the most critical rating, which reduces "problem" words.
      If both incoming and existing words are auto-loaded, the higher rating will prevail. This is done to favor words from more "quality" sources.
      Rating Method:
      Once a word is hand-rated, it will remain that way. The only way to clear a hand-rated flag is to purge the database and reload data from scratch.
      The rank-loaded flag will override the auto-loaded flag. Once a word is rank-loaded it will remain that way until being hand-rated.
      Crossword Use (NYT or Any)
      A hand-rated word's setting overrides an existing word's setting.
      Otherwise, if either the existing or incoming word has either flag set, it will be (or remain) set. In other words, a word loading from a non-puzzle word list does not remove the fact that it previously came in from a puzzle word list.
      Needs Research:
      A hand-rated word's setting overrides an existing word's setting.
      Otherwise, if either the existing or incoming word has the research flag set, it will be (or remain) set.
      Comments:
      Comments are never auto-loaded. Hand-rated comments override (or remove, if deleted) existing comments.

      IMPORTANT:

      When manually adding words, from the Add screen, or loading with the "Hand-rated" flag set, the metadata settings that you select will override any existing metadata. This is because "Hand-rated" is essentially a manual override and enforcing any other logic would make it so you could never add or clear certain flags. Manual changes will always take precedence over any data currently in the database.

      This is also why loading word lists with the "Hand-rated" flag can be so dangerous, as it essentially bypasses all reconciling and treats each entry as a manual add.

  10. Exporting Word Lists

    11. Exporting is how you use your database to create dictionary files to be used with crossword construction software (e.g. Crossword Compiler or CrossFire).

    The default parameters on this screen are to export everything in the database that has at least the minimum-specified rating. Here I use 10 as the cut-off, as anything below that is way too bad to even consider. Of course, your mileage may vary, as may your rating ranges. This default value is set as EXPORT_DEFAULT_MINIMUM_RATING in the config.txt file.

    Enter the name of the file you want to create in the "File to Export" field. By default, this file will be created in the "export" directory, but this path can be manually edited (and the default path can be overridden in the config.txt file using the EXPORT_FILE_DEFAULT_DIR parameter). See Configuration (or the comments in the config.txt file itself) for more details.

    Tip:

    CrossFire dictionaries can reside anywhere, so I export my files with names in the form dictYYYYMMDD.dict (where YYYYMMDD is today's date, e.g., dict20200101.dict) and then select them using the "browse" feature in CrossFire to load them directly in. This way I can easily make rating changes and reload a new dictionary with relative ease. If I'm doing multiple iterations, I'll add an 'a', 'b', 'c', etc. to the file name to keep them unique (e.g., dict20200101a.dict).

    In Crossword Compiler you can import your exported word list from the Convert menu in the Word List Manager.

    Exported files contain one entry per line, in the format: WORD;RATING (see note, below), for example:

    ANT;60
    BEAR;70
    ...

    Note:

    If necessary, the delimiter between WORD and RATING can be changed by setting EXPORT_FILE_DELIMITER in the config.txt file. See Configuration (or the comments in the config.txt file) for more details.

  13. Database Menu

    1. Database Info

      2. The Database Info command shows your tables, and how many records are in each.

    2. Backup

      3. The backup command takes a full backup of your database and stores it in the "backups" subdirectory with the name "bkup_YYYY-MM-DD-HH-MM-SS.SSS", where "YYYY-MM-DD-HH-MM-SS.SSS" is the timestamp when the backup was done.

      IMPORTANT:

      It is HIGHLY recommended that you backup your database periodically, and especially before any major changes, such as loading a word list. By backing up first, if the merge goes awry (perhaps you had the wrong parameters set, for example) you can clear the tables and restore to your backup file.

      Note:

      Backups done while in TEST MODE will be named "bkup_TEST_yyyy-mm-dd-hh-mm-ss.sss". Any attempt to restore a TEST MODE backup file while not in TEST MODE will result in an error. This is to prevent accidental cross-contamination.

    3. Restore

      4. To restore a backup file, select "Restore" from the Database Menu, which will bring you to the "Load" tab and launch a file select dialog preset to the "backups" directory, if it exists. If the "backups" directory does not exist, it is probably because you have never created a backup and, therefore, have nothing to restore to. A restore file MUST come from the "backup" directory (to prevent accidental "restoring" of non-backup files).

      IMPORTANT:

      Do NOT click the "Load" button to do a restore. "Load" will only load (and merge) word and ratings. "Restore" recreates the entire database as it was when the backup file was created.

      Note that for a pristine restore, you should Clear Tables first so you are starting from scratch. The restore command does not delete any records.

      Note:

      Backups done while in TEST MODE will be named "bkup_TEST_YYYY-MM-DD-HH-MM-SS.SSS", where "YYYY-MM-DD-HH-MM-SS.SSS" is the timestamp of the backup. Any attempt to restore a TEST MODE backup file while not in TEST MODE will result in an error. This is to prevent accidental cross-contamination.

      Tip:

      If you truly want to restore a TEST MODE database into your production system, rename the backup file to remove the "_TEST" portion of the name.

    4. Clear Tables

      5. This command deletes ALL DATA from your tables. This is useful if you have a backup file that you want to return to.

      Important:

      Clearing tables is a destructive action. Make sure you have your data backed up first or have nothing to lose (e.g., you're still playing in TEST MODE).

      Note that since this is a destructive action the system will require you to enter a specific string into the Comment field to confirm that this action was not selected accidentally. It should be virtually impossible to take this action unintentionally.

  14. Database Schema

    15. Since the vast majority of entries likely will not have comments, comments are stored in a separate table, linked by primary key (ENTRY). This greatly reduces the size of any database with sparse comments.

  15. WORDS Table

    16. +----------------+-------------+------+-----+-------------------+-----------------------------+
| Field          | Type        | Null | Key | Default           | Extra                       |
+----------------+-------------+------+-----+-------------------+-----------------------------+
| ENTRY          | varchar(25) | NO   | PRI | NULL              |                             |
| LENGTH         | tinyint(4)  | NO   |     | NULL              |                             |
| RATING         | tinyint(4)  | NO   |     | NULL              |                             |
| USED_ANY       | tinyint(1)  | NO   |     | FALSE             |                             |
| USED_NYT       | tinyint(1)  | NO   |     | FALSE             |                             |
| NEEDS_RESEARCH | tinyint(1)  | NO   |     | FALSE             |                             |
| RANKED_LIST    | tinyint(1)  | NO   |     | FALSE             |
| MANUALLY_RATED | tinyint(1)  | NO   |     | FALSE             |                             |
| LAST_MODIFIED  | timestamp   | NO   |     | CURRENT_TIMESTAMP | on update CURRENT_TIMESTAMP |
+----------------+-------------+------+-----+-------------------+-----------------------------+
    ENTRY
    The word or phrase. Primary Key. Limited to 25 characters.
    LENGTH
    The integer length of ENTRY. While this could be calculated on the fly, storing it is more efficient for queries.
    RATING
    The integer rating value, from 0 to 100.
    USED_ANY
    Boolean. Set to 1 if flagged as "Used Any" (used in some puzzle), else 0.
    USED_NYT
    Boolean. Set to 1 if flagged as "Used NYT" (used in NY Times puzzle), else 0.
    NEEDS_RESEARCH
    Boolean. Set to 1 if flagged as "Needs Research", else 0.
    RANKED_LIST
    Boolean. Set to 1 if word came in from a ranked word list, else 0.
    MANUALLY_RATED
    Boolean. Set to 1 if word has been hand-rated, else 0.
    LAST_MODIFIED
    Timestamp when record was last modified in any way. Note that this field is retained during a Database Restore action (i.e., restore is not considered a modification).

  16. COMMENTS Table

    17. +---------+--------------+------+-----+---------+-------+
| Field   | Type         | Null | Key | Default | Extra |
+---------+--------------+------+-----+---------+-------+
| ENTRY   | varchar(25)  | NO   | PRI | NULL    |       |
| COMMENT | varchar(100) | YES  |     | NULL    |       |
+---------+--------------+------+-----+---------+-------+
    ENTRY
    The word or phrase. Primary Key. Limited to 25 characters.
    COMMENT
    A comment about the ENTRY. Limited to 100 characters.

  17. Create Statements

    18. Note:

    Database tables will be created, if necessary, on startup. It is not necessary to create the tables beforehand. The following are provided for technical detail only. 

    CREATE TABLE `WORDS` (
  `ENTRY` varchar(25) NOT NULL,
  `LENGTH` tinyint(4) NOT NULL,
  `RATING` tinyint(4) NOT NULL,

    CREATE TABLE `COMMENTS` (
